Viewport Meta Tag: Controll your mobile browsers.

Why Meta tag?

Viewport Meta tag in head tag will make designing flexible layout more controllable. Actually, you will have more control over the viewport on mobile browsers.

Meta are built from iOS.

iOS Safari assumes a webpage is 980px wide, which creates the problems zooming out allowing the users to see overall webpage but much readable. However, that is just how the mobile browsers will do. They assume if we do not give them appropriated instructions.

The Viewport meta tag.

< meta name = "viewport" content = "initial-scale=1" >

The content value can be vary separated by commas. For example, if our mobile design is purposely laid out at 320px. We can then set content equal to width 320px. We can then set viewport based on width-on-devices for good practic.

The width property controls the size of the viewport. It can be set to a specific number of pixels like width=600 or to the special value device-width value which is the width of the screen in CSS pixels at a scale of 100%. (There are corresponding height and device-height values, which may be useful for pages with elements that change size or position based on the viewport height.)

The initial-scale property controls the zoom level when the page is first loaded. The maximum-scale, minimum-scale, and user-scalable properties control how users are allowed to zoom the page in or out.

< meta name = "viewport" content = "width=device-width" >

To be extra certain we will ensure the scale upon opening using maximum-scale = n. The zoom level can be specified by using initial-scale= n.

< meta name = "viewport" content = "initial-scale=1" >
< meta name = "viewport" content = "maximum-scale=1" >

Problem?

By seeting viewport content to device-width together with initial-scale can cause problem when mobile browser is in the landscape mode. The browser will feel obliged to zoom in. Therefore, the content will exceed the screen. We can simply fix that problem by ignore the width and just use initial-scale.

IE10 ignores metatag tho.

In short, similar viewport properties are defined within CSS using the @viewport rule, instead of within the HTML. Also, this require the prefixed version of different browsers.

Web design for mobiles and tablets.

When we develop a mobile website for different devices, nearly every device has a different resolution and screen size. This can be difficult to design a website that works among all of them simultaneously. To start off it is a good idea to look at what screen size and resolutions are out there. Then we might have a better idea of how much space we have to work with.
